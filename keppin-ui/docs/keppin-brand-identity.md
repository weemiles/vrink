# keppin 브랜드 아이덴티티 가이드

- 문서 버전: v1.0
- 작성일: 2026-02-23
- 대상: 제품/디자인/마케팅/콘텐츠/개발 팀 공통

## 1) 브랜드 한 줄 정의
`keppin`은 소중한 사람과의 관계를 "기억-관리-행동"으로 이어주는 관계 관리 앱이다.

## 2) 브랜드 목적과 역할
### 미션
- 사용자가 중요한 인연을 놓치지 않고 꾸준히 연결되도록 돕는다.

### 비전
- 연락처 앱을 넘어, 개인의 관계 자산을 관리하는 기본 플랫폼이 된다.

### 제품 역할
- 흩어진 인연 정보를 한곳에 정리
- 생일/기념일, 연락 공백을 행동 가능한 형태로 알려줌
- "기억했다"에서 끝나지 않고 "지금 연락하기"로 이어지게 함

## 3) 타깃 사용자
### 핵심 타깃
- 사람을 중요하게 생각하지만 바쁜 일정으로 관계 관리가 어려운 사용자
- 생일, 기념일, 연락 주기 등을 체계적으로 관리하고 싶은 사용자

### 사용 맥락
- "연락해야지"를 자주 놓침
- 중요한 날을 챙기고 싶지만 도구가 분산됨
- 대인관계를 감정이 아닌 루틴으로 안정적으로 유지하고 싶음

## 4) 브랜드 가치 제안 (Value Proposition)
1. 정리: 인연 정보를 한 곳에 모아 구조화한다.
2. 예측: 다가오는 이벤트/연락 공백을 미리 보여준다.
3. 실행: 예약 메시지와 알림으로 실제 행동을 유도한다.
4. 신뢰: 과장 없이 명확한 문장과 안정적인 인터랙션으로 불안감을 낮춘다.

## 5) 브랜드 성격 (Personality)
- 따뜻함: 사람 중심, 관계를 소중히 다루는 태도
- 명료함: 짧고 분명한 문장, 즉시 이해 가능한 UI
- 절제됨: 과장/자극 대신 신뢰 기반 표현
- 실용성: 감성보다 실행 가능성 우선

## 6) 톤 앤 매너 (Verbal Identity)
### 기본 원칙
1. 사용자 탓 금지: 비난 대신 다음 행동 제시
2. 짧고 구체적으로: 결과 + 원인 카테고리 + 행동 순서
3. 성공 문구는 상태 변화 중심: "저장했어요", "변경했어요"
4. 모호한 표현 금지: "오류가 발생했습니다" 단독 사용 금지

### 문장 스타일
- 권장: "연락처를 저장했어요. 다시 확인해볼까요?"
- 비권장: "문제가 발생했습니다. 잠시 후 시도하세요."

## 7) 비주얼 아이덴티티 (Visual Identity)
### 로고 콘셉트
- Connection K
- `K` 형태 안에 관계의 연결선을 담아 "나-타인-관계"를 시각화
- 요소 의미
  - Stem: 나, 중심축
  - Curved arms: 관계 연결선
  - Dots: 연결된 사람

### 컬러 시스템 (TDS 연동)
- 라이트 배경: `#FFFFFF` (고정)
- 다크 배경: `#1A1E27` (고정)
- 메인 컬러: `#1A1E27`
- Positive: `#21C389`
- Negative: `#FF4D4D`
- Sub Green: `#34C185`
- Sub Blue: `#3F80FF`
- Sub Pink: `#FE88E4`
- 중립 그레이 스케일: `#F4F6F8` ~ `#1A1E27`

### 컬러 사용 비율 가이드
- Neutral 70 / Main 20 / Accent 10
- 기본 화면은 중립색 우선, 강조는 핵심 액션/상태에만 사용

## 8) 제품 경험 원칙 (UX Identity)
1. 작은 인지 부하: 한 화면 핵심 행동 1개 중심
2. 즉시 행동 가능: 알림/리마인더는 항상 "다음 액션"을 동반
3. 모바일 실사용 기준: 8pt 그리드, 최소 터치 44px, 안전영역 준수
4. 신뢰 우선: 읽기 쉬운 계층, 무리한 모션/과한 장식 지양

## 9) 시그니처 경험
### 온보딩
- "소중한 사람 관리 -> 중요한 날 리마인드 -> 연락 유지" 순으로 가치 인지

### 홈
- 오늘 필요한 관계 행동을 우선 노출
- 카드/리스트 기반으로 "누구에게, 왜, 언제"가 바로 보이게 구성

### 메시지
- 관계 맥락(생일/기념일/공백)에 맞는 커뮤니케이션 실행을 지원

## 10) 브랜드 적용 Do / Don't
### Do
- 명확한 행동 문구 사용
- 감정적으로 따뜻하되 정보는 구체적으로 전달
- 색상은 역할 중심으로 사용 (의미 없는 장식색 남용 금지)

### Don't
- "최종/긴급/반드시" 같은 과장형 카피 남용
- 사용자 실수로 몰아가는 에러 문구
- 중요한 액션 외 영역에 강한 포인트 컬러 과다 사용

## 11) 운영 체크리스트
1. 카피가 브랜드 톤 원칙(비난 금지/행동 제시/짧은 문장)을 지키는가
2. 컬러가 Neutral 70/Main 20/Accent 10 범위를 벗어나지 않는가
3. 로고/아이콘/타이포가 TDS 토큰 체계와 일치하는가
4. 핵심 플로우에서 사용자가 다음 행동을 즉시 이해할 수 있는가
