    - TDS 전체 헌법(레이아웃/타이포/컬러/기본 컴포넌트 수치): TDS 헌법 — Toss 모바일 앱 디자인 실무 가이드라인 (2026): 레이아웃·타이포·컬러·컴포넌트·접근성
    - Overlay/Toast/Asset/Skeleton/TableRow: TDS 심화 학습 — 오버레이·토스트·에셋·스켈레톤·TableRow (원문 구조 기반)

- 각 항목은 아래 라벨로 구분합니다.

## 1) Tooltip 운영 규칙 (정보·검증·가이드)

### 1.1 언제 Tooltip을 쓰는가
- [운영 가드레일] Tooltip은 “행동을 요구하지 않는” 보조 정보에만 사용한다.
- [운영 가드레일] 사용자의 결정을 바꾸는 핵심 정보면 Tooltip이 아니라 Dialog/BottomSheet로 올린다.

### 1.2 노출 빈도/반복 제한
- [운영 가드레일] 동일 Tooltip 자동 노출은 세션당 1회.
- [운영 가드레일] 사용자가 닫은 Tooltip은 7일 동안 자동 재노출 금지.

### 1.3 길이/형식(수치)
- [운영 가드레일] 1줄 권장: 20자 이내(한글 기준).
- [운영 가드레일] 2줄 상한: 40자 이내.
- [운영 가드레일] 3줄이 필요하면 Tooltip을 포기하고 “도움말 페이지/바텀시트”로 분리한다.

### 1.4 접근성
- [운영 가드레일] Tooltip 트리거는 터치 타겟 44×44px 이상을 보장한다(최소 터치 규칙과 동일).[1]
- [운영 가드레일] 키보드/스크린리더 포커스로도 열리고 닫히는 흐름을 제공한다.

---

## 2) Empty / Error / Retry 상태 모델(페이지·섹션 공통)

### 2.1 상태 타입(고정)
- [운영 가드레일] 모든 데이터 화면은 아래 5가지 상태를 반드시 가진다.
    1) loading
    2) success(hasData)
    3) success(empty)
    4) error(retryable)
    5) error(non-retryable)

### 2.2 재시도 UX 규칙(수치)
- [운영 가드레일] 사용자가 누르는 “재시도”는 즉시 시각 피드백(200ms 이내)이 있어야 한다.
- [운영 가드레일] 자동 재시도(backoff): 최대 2회.
    - 1회: 1초
    - 2회: 3초
    - 이후는 사용자 재시도 버튼으로 전환

### 2.3 에러 메시지 규격(수치)
- [운영 가드레일] Title: 15자 이내, Description: 60자 이내.
- [운영 가드레일] 해결 가능한 에러는 Description에 “다음 행동”을 1개만 제시한다.

### 2.4 로그(관측) 최소 스키마
- [운영 가드레일] error_code, endpoint, latency_ms, retry_count, screen_id는 반드시 남긴다.

---

## 3) Input(텍스트 입력) 운영 규칙 — 검증·마스킹·키보드

### 3.1 검증 타이밍 규칙
- [운영 가드레일] 입력 중(typing)에는 “형식 오류”를 즉시 띄우지 않는다.
- [운영 가드레일] 오류 노출 타이밍은 아래 중 1개로 통일한다.
    - onBlur
    - 제출 시(submit)

### 3.2 길이 제한(수치)
- [운영 가드레일] 단일 필드 기본 maxLength는 50자를 기본값으로 검토한다(특수 케이스는 별도).
- [운영 가드레일] 메모/긴 입력(TextArea)은 500자 상한을 기본으로 둔다.

### 3.3 마스킹 규칙(PII)
- [운영 가드레일] 주민번호/계좌/카드 등 민감값은:
    - 화면 표시: 마지막 4자리만 노출
    - 복사: 기본 비활성, 필요한 경우 별도 확인

### 3.4 키보드/포커스(모바일)
- [운영 가드레일] 다음 필드로 이동은 returnKeyType=next 같은 “연속 입력” 경험을 기본으로 제공한다.
- [운영 가드레일] 폼 제출 버튼은 키보드로 가려지지 않게 한다(아래 BottomCTA 규칙과 연결).

---

## 4) BottomCTA(하단 고정 CTA) 운영 규칙

### 4.1 Single vs Double 선택 기준
- [운영 가드레일] Primary CTA는 항상 1개를 기본으로 설계한다.
- [운영 가드레일] Double은 아래 조건에서만 허용한다.
    - 취소가 “같은 중요도”로 필요한 고위험 액션(결제/삭제/해지)
    - 또는 좌/우로 의미가 완전히 분리되는 흐름(이전/다음)

### 4.2 버튼 문구(수치)
- [운영 가드레일] CTA 문구는 최대 8자(한글).
- [운영 가드레일] 9자 이상이면 문구를 줄이거나 2단 구조(상단 설명 + 버튼 짧게)로 바꾼다.

### 4.3 스크롤 숨김 정책
- [운영 가드레일] 긴 스크롤 화면에서 CTA가 시야를 계속 가리면:
    - hideOnScroll을 기본 검토
    - 다시 노출은 “위로 스크롤”에서만

### 4.4 키보드 위 고정(입력 화면)
- [운영 가드레일] 입력 화면에서 CTA가 핵심이면 “키보드 위 고정”을 기본 원칙으로 한다.
- [운영 가드레일] 키보드 노출 시 레이아웃 점프를 막기 위해:
    - 화면 전환 애니메이션과 겹치지 않게
    - CTA 영역 높이는 고정(가변 높이 금지)

## 5) Highlight(튜토리얼/가이드) 운영 규칙

### 5.1 노출 정책(수치)
- [운영 가드레일] 신규 사용자 온보딩 하이라이트는 3스텝 이내.
- [운영 가드레일] 동일 하이라이트 자동 노출은 사용자당 1회.
- [운영 가드레일] 이후는 설정에서 “다시 보기”로만 제공.

### 5.2 차단/스킵
- [운영 가드레일] 스킵 버튼을 항상 제공한다.
- [운영 가드레일] 스킵을 선택하면 해당 가이드는 30일간 자동 재노출 금지.

---

## 6) Chart(BarChart 등) 운영 규칙 — 해석/축/데이터 품질

### 6.1 데이터 포인트 수(수치)
- [운영 가드레일] 막대/점이 12개를 초과하면:
    - 기간 필터(주/월/분기) 또는 스크롤/줌 제공
    - 또는 Top N + 나머지 묶기

### 6.2 축/라벨
- [운영 가드레일] Y축 라벨은 0 포함을 기본으로 한다(오해 방지).
- [운영 가드레일] 값 단위(원/명/% 등)는 제목 또는 축에 반드시 1회 표기한다.

### 6.3 빈 데이터
- [운영 가드레일] 0이 의미 있는 경우(“실적 0”)와 데이터 없음(“집계 전”)을 분리한다.
    - 0 vs null을 의미적으로 분리

---

## 7) 적용 체크리스트 (빠른 점검)
[ ] Tooltip 자동 노출은 세션당 1회, 닫으면 7일 재노출 금지
[ ] Empty/Error 상태는 5종으로 고정하고 retry 자동 2회까지만
[ ] Input 검증은 onBlur 또는 submit 중 하나로 통일
[ ] BottomCTA는 기본 Single, Double은 고위험/명확한 분기만
[ ] Highlight는 3스텝 이내, 사용자당 1회, 스킵 시 30일 차단
[ ] Chart는 12포인트 넘으면 축약/필터/스크롤 중 하나 적용
