## 1) 입력/폼(Form) 심화 — 검증·에러·마스킹·키보드 운영 규칙

### 1.1 검증 타이밍(Validation timing)
- [운영 가드레일] 입력 중(keystroke) 검증은 기본 OFF.
    - 예외: 포맷팅(전화번호/금액)처럼 실시간 보정이 UX에 도움이 되는 필드
- [운영 가드레일] 에러 노출 타이밍 기본 규칙
    - 1차: blur(포커스 아웃)에서 노출
    - 2차: submit에서 모든 필드 강제 노출
- [운영 가드레일] 서버 검증(중복/존재 여부)은 디바운스 후 호출
    - 디바운스: 500ms
    - 타임아웃: 3초
    - 실패 시: 재시도 버튼 제공, 자동 재시도는 최대 1회

### 1.2 에러 메시지 표준
- [운영 가드레일] 에러 메시지는 1줄을 목표로 하고, 최대 2줄까지만 허용
- [운영 가드레일] 에러 메시지 구성 순서 고정
    1) 무엇이 문제인지(원인)
    2) 무엇을 해야 하는지(행동)
- [운영 가드레일] 동일 에러는 같은 문구로 매핑(제품 전역)

### 1.3 입력 길이/허용 문자 규칙(기본형)
- [운영 가드레일] 텍스트 입력 기본 상한
    - 이름/닉네임: 2~20자
    - 한 줄 설명: 최대 40자
    - 상세 설명(멀티라인): 최대 500자
- [운영 가드레일] 금칙/제어문자 제거
    - 앞뒤 공백 trim
    - 연속 공백은 1개로 축약

### 1.4 마스킹/표기(PII)
- [운영 가드레일] 화면 표시 마스킹 규칙
    - 이메일: ab***@domain.com(로컬 파트 2글자 + * 고정)
    - 전화번호: 010-****-1234(중간 4자리 마스킹)
    - 계정/식별자: 뒤 4자리만 노출
- [운영 가드레일] 클립보드 복사는 “민감 정보”에서 기본 OFF.
    - 예외 허용 시: 복사 후 3초 내 토스트로 안내(복사됨) + 앱 백그라운드 전환 시 클립보드 삭제(가능하면)

### 1.5 키보드/포커스 운영
- [운영 가드레일] 폼 제출 실패(검증 에러) 시
    - 첫 번째 에러 필드로 스크롤 + 포커스 이동
    - 이동 애니메이션 시간: 200~300ms
- [운영 가드레일] 엔터(Next/Done) 흐름
    - 다음 입력으로 이동은 최대 3단계까지만 연속 허용
    - 4번째부터는 화면 스크롤/키보드 가림 이슈가 커지므로, 섹션 분리 또는 단계형으로 전환

## 2) 선택 컨트롤(Selection) — 상태/복원/비활성 규칙

### 2.1 단일/복수 선택 상한
- [운영 가드레일] 복수 선택(Multi-select) 상한을 반드시 명시
    - 기본 상한: 최대 3개
    - 상한 도달 시: 비활성 + 안내(“최대 3개까지 선택할 수 있어요”)

### 2.2 기본값/복원 규칙
- [운영 가드레일] 선택 UI는 항상 3가지 상태를 가진다.
    - default(초기값)
    - current(사용자 변경)
    - saved(저장된 값)
- [운영 가드레일] ‘초기화’가 있는 화면이면 1탭으로 되돌리기
    - 되돌리기 대상: current → saved
    - “전체 초기화”는 확인 다이얼로그(Confirm) 필수

### 2.3 비활성(Disabled) vs 잠금(Locked)
- [운영 가드레일] disabled는 “선택 불가”만 의미.
    - 왜 불가인지 설명이 필요하면 잠금(locked) 패턴 사용
- [운영 가드레일] locked는 클릭 가능
    - 클릭 시 사유 안내 + 해결 행동(예: 본인인증 필요)
    - 잠금 안내는 같은 세션에서 최대 2회까지만 강제 노출(그 이후는 토스트/헬프 링크로 축소)

## 3) Tooltip / Popover / Coachmark — 노출 빈도·완료 처리 운영

### 3.1 노출 빈도 제한(Frequency capping)
- [운영 가드레일] 코치마크/가이드(강제 노출)는
    - 기능/화면당 최대 1회/일
    - 동일 사용자 기준 최대 3회 노출 후 자동 중단

### 3.2 완료(learned) 조건
- [운영 가드레일] 완료 조건은 2가지 중 택1로 고정
    1) 사용자가 ‘확인’ 버튼을 눌렀다.
    2) 해당 기능을 실제로 1회 수행했다(행동 기반)
- [운영 가드레일] 완료된 가이드는 재노출 금지. (단, 기능이 바뀐 major 업데이트는 예외)

### 3.3 화면 방해 레벨 기준
- [운영 가드레일] Tooltip(가벼움)
    - 자동 노출 가능, 자동 닫힘: 3~5초
- [운영 가드레일] Coachmark(강함)
    - 자동 노출 금지(사용자 트리거 기반) 또는 1회만
    - 닫힘은 사용자가 직접 수행(탭/확인)

## 4) Empty / Error / Offline / Retry — 표준 상태 모델(상태 머신)

### 4.1 화면 상태 정의(최소 6상태)
- [운영 가드레일] 모든 데이터 화면은 아래 6상태 중 하나여야 한다.
    - loading_initial
    - loaded
    - empty
    - error_retryable
    - error_blocking
    - offline_stale(캐시 표시)

### 4.2 Retry 규칙(네트워크)
- [운영 가드레일] 자동 재시도는 최대 2회
    - 백오프: 1s → 3s
- [운영 가드레일] 2회 실패 후에는 수동 재시도 버튼만
- [운영 가드레일] 연속 실패 3회 이상이면
    - 오류 원인 추정 문구 노출
    - 문의/신고 경로(또는 상태 페이지) 제공

### 4.3 오프라인 캐시 표시 규칙
- [운영 가드레일] 캐시 데이터로 화면을 그릴 때는 “최신 아님”을 표시
    - 기준: 마지막 동기화로부터 5분 초과면 stale 배지 노출
    - 30분 초과면 상단에 고정 배너(동기화 필요)로 승격

## 5) 데이터 표시(Data Display) — 배지/태그/테이블/페이지네이션 운영 규칙

### 5.1 숫자 표기(Formatting) 기본
- [운영 가드레일] 금액
    - 1,000 단위 구분(로케일 적용)
    - 소수점은 기본 숨김, 예외는 2자리까지
- [운영 가드레일] 퍼센트
    - 기본 0~100% 범위
    - 1% 미만은 <1%로 표기

### 5.2 테이블/리스트의 “한 화면 정보량” 제한
- [운영 가드레일] 한 화면(1 view)에서
    - 리스트는 최대 30행까지만 즉시 렌더
    - 그 이상은 페이지네이션 또는 무한 스크롤

### 5.3 페이지네이션 기본값
- [운영 가드레일] 페이지 크기(page size): 20
- [운영 가드레일] “더 보기” 버튼 전환 기준
    - 3페이지 이상 연속 스크롤이 필요하면 “더 보기”로 전환 검토

### 5.4 상태 배지(Badge) 운영
- [운영 가드레일] 상태는 최대 5개로 제한
    - 예: active / pending / failed / blocked / done
- [운영 가드레일] 색상(semantic) 의미를 고정
    - 성공=green, 경고=orange, 실패=red, 중립=grey

## 6) 차트/시각화(Chart) — 해석 가드레일(오해 방지)

### 6.1 축/단위/반올림 규칙
- [운영 가드레일] 모든 차트는 단위(unit)를 반드시 표기
    - 원, %, 건, 명, 시간 등
- [운영 가드레일] 반올림 규칙
    - 큰 수(>= 10,000): 천 단위 반올림(예: 12,345 → 12.3k)
    - 퍼센트: 기본 소수점 0자리, 변동이 작으면 1자리

### 6.2 결측/이상치 처리
- [운영 가드레일] 결측(null)은 0으로 치환하지 않는다.
    - 선형 차트는 끊김으로 표시
    - 집계는 “데이터 없음”으로 분리
- [운영 가드레일] 이상치(outlier)로 스케일이 무너지면
    - p95 상한으로 클리핑한 뷰를 보조로 제공(토글)

### 6.3 비교/증감 문구의 안전 규칙
- [운영 가드레일] 증감(+X%)은 표본 수가 적으면 숨김
    - 기준: 분모가 100 미만이면 증감 문구 숨김(오해 방지)
- [운영 가드레일] 실험/변경 전후 비교는 기간을 최소 7일 이상 맞춘다.

## 7) 최소 적용 순서(빠른 실무 도입)
1) 폼 검증 타이밍 + 서버 검증 디바운스(1.1) 고정
2) 선택 컨트롤 상한(2.1) + locked/disabled 구분(2.3)
3) 코치마크 노출 캡(3.1) 먼저 넣기
4) 화면 상태 6종(4.1)으로 화면 분류
5) 리스트 30행 제한 + 페이지 size 20(5.2~5.3)
6) 차트 단위/결측/표본수 가드레일(6.1~6.3)
