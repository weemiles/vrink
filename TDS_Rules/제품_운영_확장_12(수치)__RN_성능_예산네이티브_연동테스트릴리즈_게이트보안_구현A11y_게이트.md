## 1) React Native 성능 예산(Performance Budget)

### 1.1 프레임/응답성 목표
- [운영 가드레일] 사용자 입력(탭/스크롤/전환)에 대한 첫 시각 피드백: 100ms 이내.
- [운영 가드레일] 스켈레톤/로더 등 로딩 표시 등장: 200ms 이내.
- [운영 가드레일] 화면 전환 애니메이션이 있는 경우: 400ms 이내로 종료(핵심 흐름 기준).

### 1.2 리스트(FlatList/SectionList) 기본 규칙
- [운영 가드레일] “스크롤 가능한 리스트”는 원칙적으로 가상화 리스트를 사용한다.
- [운영 가드레일] 한 화면에서 “동시에” 렌더링되는 셀(아이템) 수는 12개 이하를 목표로 한다.
    - 초과 시: placeholder(스켈레톤) + 점진 로딩 + 이미지 디코딩 지연 전략 적용
- [운영 가드레일] key는 index를 쓰지 않고, 안정적 id를 사용한다.

### 1.3 이미지/미디어 비용
- [운영 가드레일] 목록 썸네일은 1) 고정 크기 2) 동일 종횡비로 강제한다.
- [운영 가드레일] 상세 화면 상단 히어로 이미지는:
    - 로딩 실패 시 fallback(컬러 블록/기본 이미지)을 반드시 제공
    - 다운로드 실패 재시도는 최대 2회

### 1.4 애니메이션/제스처
- [운영 가드레일] Reduce Motion 환경에서는:
    - 모션 제거 또는 duration을 50% 이하로 축소
    - 정보 전달이 모션에 의존하지 않게 한다

## 2) RN New Architecture / 네이티브 연동 운영 규칙

### 2.1 브릿지/네이티브 모듈 도입 기준
- [운영 가드레일] 아래 중 하나면 네이티브 모듈/컴포넌트 검토
    - JS만으로 성능 목표(1.1)를 충족하기 어려움
    - OS 권한/시스템 기능을 직접 써야 함
    - 대용량 미디어/실시간 처리가 필요

### 2.2 네이티브 이벤트/로그 규칙
- [운영 가드레일] 네이티브→JS 이벤트는 최소 스키마를 강제한다.
    - event_name, timestamp_ms, screen_id, session_id
- [운영 가드레일] 이벤트 payload 크기 상한: 4KB
    - 초과 시: 파일/저장소 참조 키로 전환

## 3) 테스트 전략 + 릴리즈 게이트(수치)

### 3.1 테스트 피라미드(비율)
- [운영 가드레일] 신규 기능 PR은 테스트 비율을 아래로 맞춘다.
    - 단위(Unit): 70%
    - 통합(Integration): 20%
    - E2E: 10%

### 3.2 커버리지(최소 기준)
- [운영 가드레일] 핵심 도메인 로직(결제/정산/인증/권한)은 라인 커버리지 80% 이상.
- [운영 가드레일] UI 컴포넌트는 커버리지보다 “상태 수”로 게이트한다.
    - 각 화면은 최소 3개 상태 테스트: success / empty / error

### 3.3 릴리즈 전 필수 시나리오
- [운영 가드레일] 릴리즈마다 아래는 최소 1회 실기기 점검
    - 로그인/가입
    - 핵심 전환(결제 또는 핵심 CTA)
    - 설정
    - 오류/오프라인(네트워크 차단)

### 3.4 크래시/에러 예산
- [운영 가드레일] 릴리즈 후 24시간 내
    - 크래시 프리 세션: 99.5% 이상
    - 주요 API 에러율: 1% 미만
    - 초과 시: 롤백 또는 핫픽스 우선순위 1

## 4) 보안(구현)·개인정보 ‘기술 운영’

### 4.1 클라이언트 저장소 규칙
- [운영 가드레일] 토큰/세션/리프레시 토큰은:
    - OS 보안 저장소(Keychain/Keystore) 사용
    - AsyncStorage/평문 파일 저장 금지

### 4.2 로그/분석 PII 차단 룰
- [운영 가드레일] 로그/이벤트에 아래 패턴이 들어가면 업로드 차단
    - 이메일 형태 문자열
    - 전화번호 형태 문자열
    - 주민/계좌/카드로 오인될 수 있는 연속 숫자(예: 12자리 이상)
- [운영 가드레일] 마스킹 규칙: 마지막 4자리만 허용

### 4.3 레이트리밋(클라이언트 가드)
- [운영 가드레일] 인증번호 요청/재전송:
    - 1분 내 최대 1회
    - 10분 내 최대 3회
    - 초과 시 10분 쿨다운

## 5) 접근성(A11y) ‘게이트’ 운영

### 5.1 터치/탭 타겟
- [TDS 명시/운영 가드레일] 최소 터치 타겟 44×44px 유지.[1]

### 5.2 스크린리더 점검 게이트(필수)
- [운영 가드레일] 릴리즈마다 아래 5개 화면은 스크린리더(VoiceOver/TalkBack)로 점검
    - 로그인
    - 결제/핵심 전환
    - 오류 화면
    - 리스트(스크롤)
    - 설정

### 5.3 색 대비
- [운영 가드레일] 텍스트 대비는 최소 4.5:1(일반 텍스트), 가능하면 7:1을 목표로 한다.
