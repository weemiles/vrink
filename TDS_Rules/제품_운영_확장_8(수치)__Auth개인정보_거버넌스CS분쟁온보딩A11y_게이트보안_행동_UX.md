## 1) 계정/인증(Auth) UX 운영 규칙 — 가입·로그인·세션·복구

### 1.1 로그인 실패/잠금 정책(수치)
- [운영 가드레일] 연속 실패 횟수:
    - 비밀번호: 5회 실패 시 10분 잠금
    - OTP/인증번호: 3회 실패 시 5분 쿨다운
- [운영 가드레일] 잠금 상태에서는:
    - 남은 시간(분:초) 표시
    - “비밀번호 재설정/계정 찾기” 단일 CTA 제공

### 1.2 인증번호(OTP/SMS/Email) 재전송 규칙(수치)
- [운영 가드레일] 재전송 버튼 활성화: 60초 후
- [운영 가드레일] 재전송 최대: 하루 5회
- [운영 가드레일] 코드 만료: 5분

### 1.3 세션 만료/재인증 UX(규칙)
- [운영 가드레일] 세션 만료 시:
    - 현재 작업을 잃지 않도록 임시 저장 후 로그인으로 이동
    - 로그인 성공 후 원래 화면으로 1회 자동 복귀
- [운영 가드레일] 민감 행동(결제/비밀번호 변경/계정 변경)은:
    - 마지막 인증이 10분 초과면 재인증 요구

### 1.4 계정 복구(비밀번호/이메일/전화) 최소 정보 원칙
- [운영 가드레일] 복구 플로우에서 입력 필드 수:
    - 1단계당 최대 2개
    - 3개 이상이면 단계 분리

## 2) 개인정보/데이터 거버넌스 UX — 수집·동의·보관·삭제

### 2.1 동의(Consent) 화면 구조 규칙
- [운영 가드레일] 체크박스 구조:
    - 필수/선택을 섹션으로 분리
    - ‘전체 동의’는 제공하되, 필수와 선택을 시각적으로 분리(오해 방지)

### 2.2 동의 철회/설정 경로(수치)
- [운영 가드레일] 사용자가 설정에서 동의 상태를 찾기까지:
    - 탭 이동 포함 3스텝 이내

### 2.3 PII 마스킹 규칙(표기)
- [운영 가드레일]
    - 이메일: 로컬 파트 1~2자만 남기고 마스킹 (예: mi***@domain.com)
    - 전화번호: 끝 4자리만 노출 (예: 010-****-1234)
    - 계좌/카드: 끝 4자리만 노출

### 2.4 데이터 다운로드/삭제 요청 SLA 표기(수치)
- [운영 가드레일]
    - 다운로드 준비: 최대 24시간
    - 삭제 처리: 최대 7일
    - 요청 접수 즉시 상태: “접수됨”

## 3) 고객지원(CS)·문의·분쟁 UX 운영 규칙

### 3.1 문의 카테고리 수 상한(수치)
- [운영 가드레일] 1차 카테고리: 최대 8개
- 9개 이상이면:
    - 검색 가능한 카테고리 선택 UI로 전환

### 3.2 문의 폼 최소 입력 정책
- [운영 가드레일] 필수 입력은 최대 3개
    - 문의 유형
    - 문의 내용
    - 연락 수단(또는 계정 기반이면 생략)

### 3.3 증빙 업로드 규칙(수치)
- [운영 가드레일]
    - 이미지 최대 5장
    - 장당 최대 10MB (미디어 기준과 일치)
    - 파일명에 개인정보가 들어갈 수 있으므로 업로드 후 서버에서 파일명 재작성

### 3.4 분쟁(환불/정정) 상태 표준(최소)
- [운영 가드레일]
    - 접수됨 → 검토 중 → 추가 정보 요청 → 처리 완료 / 반려
- 상태 변경 시:
    - 푸시/알림은 상태 변화당 1회

## 4) 온보딩/가이드/교육 UX 운영 규칙

### 4.1 튜토리얼 길이 상한(수치)
- [운영 가드레일] 첫 실행 튜토리얼 단계:
    - 최대 5단계
    - 6단계 이상이면 “스킵”을 기본 노출

### 4.2 가치 도달(Activation) 정의 규칙
- [운영 가드레일] 온보딩은 “기능 설명”이 아니라 “가치 행동 1회 완료”까지 설계
    - 예: 저장 1회, 예약 1회, 결제 1회 등

### 4.3 반복 실패 기반 도움말 노출(수치)
- [운영 가드레일] 같은 플로우에서 실패가 2회 연속 발생하면:
    - 도움말(FAQ/가이드) 링크 1개를 추가 노출

## 5) 접근성(A11y) 운영/조직 레벨 규칙 — 릴리즈 게이트 포함

### 5.1 릴리즈 게이트(최소 체크리스트)
- [운영 가드레일] 릴리즈 전 아래 항목은 ‘차단 버그’로 처리
    - 핵심 플로우에서 키보드 포커스가 갇힘
    - 스크린리더로 CTA에 접근 불가
    - 텍스트 대비 기준(4.5:1) 미달
    - Reduce Motion에서 기능이 깨짐

### 5.2 A11y 버그 우선순위(규칙)
- [운영 가드레일]
    - P0: 결제/인증/설정에서 진행 불가
    - P1: 주요 화면에서 정보 파악 불가
    - P2: 불편하지만 진행 가능

### 5.3 스크린리더 QA 범위(수치)
- [운영 가드레일] 최소 다음 화면은 매 릴리즈마다 점검
    - 로그인/가입
    - 결제(또는 핵심 전환)
    - 설정
    - 오류/빈 상태

## 6) 보안 ‘표현/행동’ UX — 피싱/권한/민감행동

### 6.1 민감 행동 재인증 트리거(수치)
- [운영 가드레일] 아래 행동은 마지막 인증이 10분 초과면 재인증 요구
    - 결제/송금
    - 비밀번호 변경
    - 계정 이메일/전화 변경

### 6.2 디바이스 변경/이상 로그인 알림 규칙
- [운영 가드레일]
    - 새 디바이스 로그인 감지 시: 알림 1회 + “내가 한 활동이 아니에요” CTA 제공
    - 알림 메시지에는 민감정보(전체 이메일/전화/계좌)를 포함하지 않음

### 6.3 클립보드/스크린샷 리스크 안내(규칙)
- [운영 가드레일]
    - 인증번호/계좌번호 복사 시: “클립보드에 복사됨” + 30초 후 자동 삭제(가능한 플랫폼에서)
    - 스크린샷 차단이 불가하면: 촬영 시 워터마크(날짜+앱명) 또는 경고 토스트

## 7) 최소 실행 경로(적용 순서)
1) Auth: 실패/잠금(5회/10분), 재전송(60초), 코드 만료(5분)부터 고정한다.
2) 개인정보: 동의 구조(필수/선택 분리)와 마스킹 규칙을 고정한다.
3) CS: 카테고리 8개 상한, 필수 필드 3개 상한, 증빙 5장 규칙을 적용한다.
4) 온보딩: 5단계 상한 + 실패 2회 시 도움말 노출을 적용한다.
5) A11y: 릴리즈 게이트(포커스/CTA/대비/Reduce Motion)와 QA 범위를 고정한다.
6) 보안: 민감 행동 재인증(10분)과 새 디바이스 알림 UX를 고정한다.
