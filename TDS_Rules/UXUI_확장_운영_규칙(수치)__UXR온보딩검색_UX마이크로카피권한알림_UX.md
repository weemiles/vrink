## 1) UXR / 사용성 테스트(UT) 운영 규격

### 1.1 목표별 최소 샘플 수
- [운영 가드레일] 탐색형(문제 발견): 세그먼트당 5명
- [운영 가드레일] 벤치마크(현재안 품질 측정): 세그먼트당 10명
- [운영 가드레일] A/B 비교(정량 결론 목적): 안(A,B) 각각 세그먼트당 ≥ 20명 없으면 “정량 결론” 금지

### 1.2 세션 길이/과제 수
- [운영 가드레일] 1회 세션 길이: 45분(최대 60분)
- [운영 가드레일] 과제 수: 최대 6개
- [운영 가드레일] 과제 1개 목표 시간: 3~7분

### 1.3 힌트(Help) 제공 규칙
- [운영 가드레일] 힌트는 과제당 최대 2회
    - 1차: “목표 재진술”(정답 경로를 말하지 않기)
    - 2차: “정보 위치 힌트”(버튼명/레이블은 말하지 않기)
- [운영 가드레일] 2차 힌트 이후 2분 내 진전이 없으면 과제 종료 처리

### 1.4 기록/산출물 최소 포맷
- [운영 가드레일] 각 과제는 아래 4개를 반드시 기록
    - 성공 여부(성공/실패/부분 성공)
    - 소요 시간(초)
    - 오류/되돌림 횟수(0,1,2…)
    - 발화 1줄 요약(불편/혼란의 원인)
- [운영 가드레일] 결과는 48시간 내 정리(기억 오염 방지)

## 2) 리서치 인사이트 → UI 결정 연결 규칙

### 2.1 “관찰”과 “해석/결정” 분리
- [운영 가드레일] 리서치 문서에는 반드시 구분 섹션을 둔다.
    - Observation: 사실(사용자 행동/발화)
    - Interpretation: 원인 가설
    - Decision: 무엇을 바꾸는지

### 2.2 우선순위 점수(간단 수치화)
- [운영 가드레일] 이슈는 아래 점수로 정렬한다.
    - Severity(심각도) 1~5
    - Frequency(빈도) 1~5
    - Effort(해결 비용) 1~5
- 추천 계산: Priority = Severity × Frequency ÷ Effort
- [운영 가드레일] Priority 상위 3개만 다음 스프린트에 반영(산만함 방지)

## 3) 온보딩(학습) UX — 퍼널/정보량 규칙

### 3.1 온보딩 단계 정의(최소)
- [운영 가드레일] 온보딩을 3단계로 고정해 정의한다.
    1) 첫 실행(Why)
    2) 첫 설정(Enable)
    3) 첫 성공 경험(Win)

### 3.2 정보량 제한
- [운영 가드레일] 1스크린에서 사용자가 해야 할 ‘주 행동’은 1개만 남긴다.
- [운영 가드레일] 온보딩 화면의 본문 텍스트는 2줄(약 40자) 이내를 기본으로 한다.
    - 초과하면 “자세히 보기”로 분리

### 3.3 스킵/나중에(우회) 정책
- [운영 가드레일] ‘나중에’가 가능한 단계는 반드시 우회 버튼 제공
    - 단, 보안/결제/필수 권한은 예외
- [운영 가드레일] 우회한 항목은 이후 진입 시 최대 2회만 재노출
    - 2회 이후는 설정/도움말로만 안내

## 4) Progressive Disclosure(점진적 공개)

### 4.1 고급 옵션 노출 조건
- [운영 가드레일] 아래 중 1개 이상 충족 시에만 고급 옵션을 기본 노출
    - 전체 사용자의 ≥ 30%가 월 1회 이상 사용
    - 고급 옵션이 없으면 핵심 과제가 불가능
- 그 외에는 “고급 설정” 토글/접기 안에 배치

### 4.2 접기 기본값
- [운영 가드레일] 고급 영역은 기본 ‘접힘’
- [운영 가드레일] 사용자가 2회 이상 펼치면 이후 기본값을 ‘펼침’으로 승격(개인화)

---

## 5) 검색 UX(컴포넌트 스펙이 아닌 행동/정책)

### 5.1 0 결과(Empty) 처리 원칙
- [운영 가드레일] 0 결과 화면은 항상 대체 행동을 최소 2개 제공
    - 예: 필터 초기화
    - 예: 추천 카테고리 보기

### 5.2 자동완성/추천 노출 수
- [운영 가드레일] 자동완성/추천은 최대 8개
- [운영 가드레일] 최근 검색어는 최대 10개, 30일 지나면 자동 삭제

### 5.3 디바운스(입력 반응)
- [운영 가드레일] 검색어 입력에 따른 API 요청은 디바운스 300ms 기본
- [운영 가드레일] 네트워크 불안정 시(타임아웃/오류 연속 2회)에는 디바운스를 600ms로 상향

## 6) UX Writing / 마이크로카피 규칙

### 6.1 에러 문장 구조(고정)
- [운영 가드레일] 에러 메시지는 아래 순서를 유지
    1) 무엇이 안 됐는지(사실)
    2) 사용자가 지금 할 수 있는 행동(1개)
    3) 필요한 경우에만 원인(추정 표현)

### 6.2 길이/문장 수
- [운영 가드레일] 토스트/짧은 피드백: 1문장, 20자 이내를 기본
- [운영 가드레일] 다이얼로그 본문: 2문장 이내

### 6.3 금지 패턴
- [운영 가드레일] 사용자 탓으로 들리는 표현 금지
    - 예: “올바르게 입력하세요”, “다시 확인하세요” 단독 사용 금지
- [운영 가드레일] 모호한 표현 금지
    - 예: “오류가 발생했습니다” 단독 사용 금지 → 다음 행동 포함

## 7) 권한(Permission) UX 운영 규칙

### 7.1 요청 타이밍
- [운영 가드레일] 권한은 앱 시작 시 일괄 요청 금지
    - 반드시 ‘기능을 쓰려는 순간’에 요청

### 7.2 거절 이후 재요청
- [운영 가드레일] 동일 권한 재요청은 최소 7일 쿨다운
- [운영 가드레일] 2회 거절 이후에는 시스템 팝업 대신
    - 설정으로 이동 CTA 1개만 제공

---

## 8) 알림(푸시/인앱) UX 운영 규칙

### 8.1 빈도 상한(기본)
- [운영 가드레일] 사용자 1인 기준
    - 프로모션/마케팅: 주 2회 상한
    - 거래/보안/필수: 상한 없음(단, 동일 이벤트 중복 발송 금지)

### 8.2 묶음/중복 방지
- [운영 가드레일] 동일 주제 알림은 10분 내 2건 이상이면
    - 마지막 알림 1건으로 병합

---

## 9) 적용 순서(현실적인 최소 루트)
1) UT 최소 규격(1.1~1.4)
2) 인사이트→결정 분리 + 우선순위 점수(2.1~2.2)
3) 검색 0결과/추천 상한(5.1~5.2)
4) 에러 카피 구조 + 금지 패턴(6.1~6.3)
5) 권한/알림 쿨다운/상한(7~8)
