# 소비자 앱 UX/UI 확장 규칙(수치) — 가격/플랜·피드·개인화·검색결과·신뢰 UI


## 0) 범위/중복 방지 선언
- 이 페이지는 Toss Design System(TDS) 문서의 설계 철학(일관성, 정보 구조의 점층적 구성, 접근성/상태 고려)을 전제로, “소비자용 앱”에서 자주 발생하는 5개 영역을 수치/고정 규칙(가드레일)로만 정리합니다.
    - TDS가이드 작성 방식(큰 구조→상세 요소→상태 변화)을 따르는 것을 기본 원칙으로 둡니다: https://toss.tech/article/toss-design-system-guide
    - TDS가 컴포넌트로 ‘품질을 강제’해 디자이너가 제품 문제에 집중하게 만든다는 전제: https://toss.tech/article/toss-design-system
- 단, 아래 규칙들은 TDS가 “컴포넌트 Props/토큰 수치”로 직접 제공하는 내용이 아니라, TDS 철학을 소비자 앱 UI 설계에 적용하기 위해 민우님이 운영 규칙 형태로 만든 확장 가드레일입니다.
    - 가격/플랜(유료화) UI
    - 피드/타임라인 UI
    - 개인화(추천/설정) UI
    - 검색 결과 UI(정책이 아니라 레이아웃/정보 구조)
    - 신뢰 UI(사용자용 활동/승인 내역)
- 이 페이지는 아래 주제들을 이미 다른 문서에 존재한다고 보고, 여기서 재작성하지 않습니다.
    - UXR 샘플/세션 규격, 인사이트→결정 연결, 우선순위 점수화
    - 온보딩 퍼널/정보량(Why/Enable/Win), Progressive disclosure
    - 검색 “정책” 수치(디바운스, 0결과 행동 수, 추천/최근검색 개수 상한)
    - 마이크로카피(에러 문장 구조, 길이 제한, 금지 패턴)
    - 권한/알림 요청 타이밍/쿨다운/빈도 캡
    - 상태 모델(loading/empty/error/offline), 재시도(backoff), 캐시
    - A11y/다크모드/i18n/QA 매트릭스/Analytics 이벤트 택소노미
- 이 페이지의 목표는 “화면을 실제로 그릴 때 필요한 UI 규칙”만, 겹치지 않게 추가하는 것입니다.

## 1) 가격/플랜(유료화) UI — 비교표/가격표 화면 가드레일

### 1.1 가격표(Plan list) 기본 구조
- [운영 가드레일] 한 화면에서 비교하는 플랜 수: 최대 3개
- [운영 가드레일] 각 플랜 카드에 노출하는 핵심 정보는 최대 6줄
    - 권장 구성(6줄 안에서 해결): 가격(1) / 결제 주기(1) / 핵심 혜택 3개(3) / 보조 안내 1개(1)
- [운영 가드레일] ‘추천 플랜’ 강조는 1개만 허용

### 1.2 플랜 비교(Comparison) 정보량 상한
- [운영 가드레일] 비교표(테이블) 행 수: 최대 10행
- [운영 가드레일] 10행을 넘기면 아래 중 하나로 분리
    1) “핵심 10개” + “전체 혜택 보기(상세)”
    2) 카테고리 탭(최대 3탭)
- [운영 가드레일] 표의 각 셀은 최대 2줄

### 1.3 가격 표기(가격/할인/체험) UI 규칙
- [운영 가드레일] 가격은 항상 하나의 기준 가격만 ‘가장 크게’ 보여준다.
    - 예: 월 결제면 월 가격을 크게. 연 결제면 연 가격을 크게.
- [운영 가드레일] 할인/체험 표기는 한 카드에서 최대 2개 배지까지만
    - 예: “7일 무료” + “연간 20% 절약”
- [운영 가드레일] 숨겨진 비용(수수료/VAT)이 있다면 가격 영역에서 1회 고지
    - 고지 문구 길이: 최대 18자(한글)

### 1.4 결제 전 확인(Review) UI 최소 세트
- [운영 가드레일] 결제 직전 화면에는 아래 4개를 반드시 다시 노출
    1) 플랜 이름
    2) 결제 주기(월/연)
    3) 결제 금액
    4) 다음 결제일(또는 갱신일)

## 2) 피드/타임라인 UI — 슬롯/중복/완독 UX 규칙

### 2.1 첫 화면(Above the fold) 정보량
- [운영 가드레일] 첫 화면에서 ‘즉시 시각적으로 구분되는 카드’ 수: 최대 6개
    - 더 많으면 시각적 과밀로 판단하고 그룹/섹션 헤더로 분리
- [운영 가드레일] 카드 1개에 노출하는 행동 버튼: 최대 1개
    - 2개가 필요하면 ‘더보기(⋯)’로 접기

### 2.2 추천/콘텐츠 중복 노출 캡
- [운영 가드레일] 같은 아이템(동일 content_id) 반복 노출 상한
    - 세션 내: 최대 1회
    - 24시간 내: 최대 2회
- [운영 가드레일] ‘이미 본 콘텐츠’는 피드에서 제거하지 말고, 아래 중 1개로만 처리
    1) 시각적으로 약하게(불투명도/채도 감소)
    2) “봄” 상태 배지 1개

### 2.3 피드 정렬 규칙(기본 1개만)
- [운영 가드레일] 기본 정렬은 한 화면에서 1개만 허용
    - 예: “추천순” 또는 “최신순”
- [운영 가드레일] 정렬/필터 컨트롤은 상단에서 최대 2개 슬롯만 사용
    - 예: (정렬 1) + (필터 1)

### 2.4 완독/재방문 UI
- [운영 가드레일] ‘저장/나중에 보기’는 모든 콘텐츠 카드에 강제하지 않는다.
    - 노출 기준: 피드 상단 첫 10개 중 최대 3개만 “저장”을 1차 노출
    - 나머지는 상세 화면에서만 제공

## 3) 개인화(추천/설정) UI — 사용자가 ‘조절 가능한 것’만 노출하기

### 3.1 개인화 조절 UI의 단계 상한
- [운영 가드레일] 개인화 설정은 사용자가 “끄거나 바꿀 수 있는” 항목만 노출
- [운영 가드레일] 설정을 찾는 단계(탭 이동 포함): 3스텝 이내

### 3.2 개인화 설명(Explainability) 최소 규칙
- [운영 가드레일] 추천 카드에 “왜 이걸 보나요?” 진입점은
    - 첫 노출 세션에서만 노출하거나
    - 상단 첫 5개 카드 중 1개에만 노출
    (둘 중 하나를 선택)
- [운영 가드레일] 설명 텍스트는 기본 1문장, 최대 2문장
    - 문장당 최대 20자(한글) 목표

### 3.3 관심 없음/차단 피드백 루프
- [운영 가드레일] 차단/관심 없음 액션은 한 카드에서 1개만 제공
    - 예: “관심 없음” 또는 “이 주제 숨기기”
- [운영 가드레일] 차단이 적용되면 즉시 피드에서 제거하지 않고, Undo 10초를 기본 제공

## 4) 검색 결과 UI — 정보 구조/스니펫/하이라이트 규칙
> 검색 ‘정책’(디바운스, 0결과 등)은 다른 문서에 있다고 가정하고, 여기서는 결과 리스트의 UI만 다룹니다.

### 4.1 결과 아이템(리스트/카드) 정보량
- [운영 가드레일] 결과 1개에 노출하는 텍스트 줄 수 상한
    - 제목: 최대 1줄
    - 보조 정보(메타): 최대 1줄
    - 스니펫(본문 일부): 최대 2줄
    - 합계: 최대 4줄

### 4.2 하이라이트(검색어 강조) 가드레일
- [운영 가드레일] 한 결과 아이템에서 하이라이트되는 단어 수: 최대 3개
- [운영 가드레일] 하이라이트는 굵게/색상 중 1가지만 사용
    - 굵게+색상 동시 적용 금지(과강조 방지)

### 4.3 결과 내 ‘바로 행동’ 노출
- [운영 가드레일] 검색 결과 리스트에서 바로 행동(저장/공유/…)
    - 화면당 최대 1종 액션만 허용
    - 액션이 2개 이상이면 상세 화면으로 이동

## 5) 신뢰 UI(사용자용) — 최근 활동/승인 내역/보안 이벤트

### 5.1 최근 활동(Recent activity) 화면의 최소 필드
- [운영 가드레일] 활동 1행은 아래 4필드만 고정
    1) 무엇(행동명)
    2) 언제(시간)
    3) 어디서(디바이스/환경: 가능하면)
    4) 결과(성공/실패)
- [운영 가드레일] 한 화면에서 노출되는 활동 타입(카테고리) 수: 최대 6개

### 5.2 민감 변경 내역 UI
- [운영 가드레일] 아래 민감 이벤트는 ‘최근 활동’에서 최소 7일 보관 노출
    - 비밀번호 변경
    - 계정 이메일/전화 변경
    - 새 기기 로그인
    - 결제 수단 변경

### 5.3 사용자 확인 UX(“내가 한 게 맞나요?”)
- [운영 가드레일] ‘내가 한 활동이 아님’ 흐름은 행동을 1개로만 고정
    - 예: “로그아웃하고 비밀번호 변경하기”
- [운영 가드레일] 해당 흐름 진입 시, 추가 정보 입력은 최대 2개 필드

---

## 6) 적용 우선순위(현실적 최소 루트)
1) 가격/플랜: 3플랜 상한 + 비교표 10행 상한
2) 피드: content_id 중복 캡(세션 1회/24시간 2회)
3) 개인화: 설정 3스텝 이내 + Undo 10초
4) 검색 결과 UI: 결과 1개 4줄 상한 + 하이라이트 3개 상한
5) 신뢰 UI: 최근 활동 4필드 고정 + 민감 이벤트 7일 노출
