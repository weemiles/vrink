## 1) 성능 엔지니어링 — 측정/예산/회귀 차단

### 1.1 성능 지표(최소 세트)
- [운영 가드레일] 화면 단위로 최소 4지표를 고정해 수집
    - App start to first screen (cold start)
    - 화면 첫 콘텐츠 표시(FCP 유사)
    - 입력 지연(탭/스크롤 반응)
    - 크래시율(세션당)

### 1.2 성능 예산(기본값)
- [운영 가드레일] 릴리즈 차단 임계치(기본)
    - cold start p95 ≤ 2.5s
    - 첫 콘텐츠 표시 p95 ≤ 1.8s
    - 입력 지연 p95 ≤ 100ms
    - 크래시율 < 0.3% / release

### 1.3 회귀 감지(비교 윈도우)
- [운영 가드레일] 비교 기준은 항상 “직전 안정 릴리즈”로 고정
- [운영 가드레일] 아래 중 1개라도 발생 시 자동 롤백 후보로 승격
    - p95 지표가 +10% 이상 악화
    - 크래시율이 +0.1%p 이상 상승

### 1.4 번들/리소스 예산
- [운영 가드레일] 앱 번들(또는 JS 번들) 증분은 릴리즈당 +5%를 상한
- [운영 가드레일] 화면에 진입 시 네트워크 요청 수는 초기 렌더 기준 ≤ 6개(권장)

## 2) 네트워크/오프라인/동기화 — 재시도/TTL/충돌

### 2.1 재시도(Backoff) 표준
- [운영 가드레일] 네트워크 실패(타임아웃/일시적 5xx) 시 자동 재시도는
    - 최대 3회
    - 지연: 0.5s → 1s → 2s (지수 백오프)
    - 요청 총 대기(추가 지연 합) ≤ 4s

### 2.2 타임아웃 표준
- [운영 가드레일] API 요청 타임아웃 상한
    - 연결+응답 합산 10s

### 2.3 캐시 TTL 기본값
- [운영 가드레일] “변경 빈도 낮은” 설정/메타 데이터 TTL
    - 24h
- [운영 가드레일] “중간 빈도” 목록/카탈로그 TTL
    - 1h
- [운영 가드레일] “민감/실시간” 잔액/권한/보안 상태 TTL
    - 0s (항상 최신 조회)

### 2.4 오프라인 모드 경계(Degraded mode)
- [운영 가드레일] 오프라인이면 아래만 허용
    - 캐시된 데이터 조회
    - 로컬 임시 저장(초안, 즐겨찾기 등)
- [운영 가드레일] 오프라인에서 서버 확정이 필요한 작업(결제/환불/권한 변경)은
    - 즉시 차단 + “연결 후 재시도”로 전환

### 2.5 동기화 충돌 해결(기본 정책)
- [운영 가드레일] 단일 필드 편집형 데이터는 기본 LWW(Last write wins)
    - 단, 서버 타임스탬프 기준
- [운영 가드레일] 리스트/컬렉션은 기본 merge + 중복 제거(key 기준)

## 3) 인증/세션/권한 — 만료/재인증/고위험 액션

### 3.1 세션 만료/갱신 규칙(기본값)
- [운영 가드레일] access token TTL: 15분
- [운영 가드레일] refresh token TTL: 30일
- [운영 가드레일] refresh 실패 시
    - 즉시 로그아웃이 아니라 “재로그인 요구” 화면으로 전환

### 3.2 동시 로그인 정책
- [운영 가드레일] 동시 로그인은 기본 허용
    - 단, 민감 기능(송금/환불/계정잠금) 수행 시 step-up auth 요구

### 3.3 Step-up 인증(재인증) 기준
- [운영 가드레일] 아래 액션은 실행 전 5분 이내 재인증이 필요
    - 환불/취소/정산 관련 최종 확정
    - 계정 비밀번호/2FA 변경
    - 권한 변경(운영자/관리자)

### 3.4 권한 UX 규칙
- [운영 가드레일] 권한 요청 팝업은 기능 진입 시점에만(앱 시작 직후 일괄 요청 금지)
- [운영 가드레일] 권한 거부 상태에서 동일 팝업을 연속 노출하지 않는다.
    - 재요청은 최소 24h 이후 또는 사용자가 설정 화면에서 명시적으로 재시도할 때

## 4) Fraud / Trust & Safety — 스코어 구간/차단/복구

### 4.1 리스크 스코어 구간(기본)
- [운영 가드레일] 0~100 스코어로 정규화
    - 0~29: Low
    - 30~69: Medium
    - 70~100: High

### 4.2 정책 매핑
- [운영 가드레일] Medium(30~69)
    - step-up auth(추가 인증) 요구
    - 거래 한도(금액/횟수) 임시 제한
- [운영 가드레일] High(70~100)
    - 거래 보류(hold)
    - 계정 잠금 후보 + 운영자 리뷰 큐로 이동

### 4.3 오탐/복구 SLA
- [운영 가드레일] 사용자 이의제기(appeal) 접수 후
    - 1차 응답 SLA: 24h
    - 잠금 해제/복구 결정 SLA: 72h

### 4.4 탐지 품질 운영
- [운영 가드레일] 차단 정책 변경은 shadow mode로 최소 7일 관측 후 적용
    - 차단 전환 시 false positive rate가 1%를 넘으면 롤백

## 5) 릴리즈 엔지니어링 — 점진 배포/플래그/자동 롤백

### 5.1 점진 배포(기본 램프)
- [운영 가드레일] 모바일 릴리즈 램프(권장)
    - 1% (2시간)
    - 10% (6시간)
    - 50% (12시간)
    - 100% (문제 없을 때)

### 5.2 피처 플래그 운영 규칙
- [운영 가드레일] 모든 신규 기능은 “코드 배포”와 “노출”을 분리
    - 기본 OFF
    - 실험/점진 노출은 서버 플래그로 제어
- [운영 가드레일] 플래그에는 만료일을 반드시 둔다.
    - 생성 시점부터 30일 내 정리(삭제/상시화)하지 못하면 운영 부채로 간주

### 5.3 자동 롤백 조건(기본)
- [운영 가드레일] 램프 단계(1%/10%/50%)마다 아래 조건 중 1개면 즉시 중단 + 롤백 검토
    - 크래시율 > 0.3%
    - 로그인 실패율 기준 대비 +0.5%p
    - 결제/핵심 액션 성공률 기준 대비 -1%p
    - API 5xx/timeout 비율 기준 대비 +1%p
