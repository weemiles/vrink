# 제품 운영 확장 6(수치) — Data Display·검색/필터·콘텐츠/미디어·i18n 길이·기능 상태모델


## 0. 범위와 중복 방지 선언 (검증용)
- 이 페이지는 컴포넌트 Props/기본값/허용값을 재작성하지 않습니다.
- 아래 기존 문서에서 다룬 내용(레이아웃/토큰/개별 컴포넌트 스펙, 오버레이/토스트/에셋/스켈레톤, 모션/딥링크/오프라인/권한/A11y/플래그)은 링크로만 참조합니다.
    - TDS 전체 헌법(레이아웃·타이포·컬러·기본 컴포넌트 스펙): TDS 헌법 — Toss 모바일 앱 디자인 실무 가이드라인 (2026): 레이아웃·타이포·컬러·컴포넌트·접근성
    - 오버레이/토스트/에셋/스켈레톤/TableRow: TDS 심화 학습 — 오버레이·토스트·에셋·스켈레톤·TableRow (원문 구조 기반)
    - SearchField/Loader/Menu/ListRow/Keypad/Carousel 등: TDS 컴포넌트 실무 노트 2 — SearchField·Loader·Menu/Dropdown·ListRow·Keypad·Carousel·NumericSpinner·BottomInfo
    - TextButton/IconButton/Result/Slider: TDS 컴포넌트 실무 노트 3(Mobile) — TextButton·IconButton·Result·Slider
    - 모션·딥링크/푸시·오프라인·권한 UX·A11y 시나리오·Server-driven UI: 제품 운영 확장 5(수치) — 모션·딥링크/푸시·오프라인·권한 UX·A11y 시나리오·Server-driven UI
- 여기서는 제품 단위 운영 규칙(수치/상한/기준)만 추가합니다.

## 1) Data display (표/리스트/카드) — 데이터 밀도/요약/상세 규칙

### 1.1 “한 화면 요약”의 데이터 밀도 상한(수치)
- 카드 1개에서:
    - 라벨-값 쌍: 최대 4개
    - 본문 텍스트: 최대 3줄
    - 서브 액션(TextButton 등): 최대 1개
- 리스트 행 1개에서:
    - 주요 정보(Primary): 1개
    - 보조 정보(Secondary): 최대 2개
    - 우측 보조(가격/상태): 최대 1개

### 1.2 요약 → 상세 분리 기준(규칙)
- 아래 중 하나라도 해당하면 “상세 보기”로 분리합니다.
    - 항목 수가 5개 초과
    - 텍스트가 4줄 이상 필요
    - 스크롤 없이 이해가 안 되는 법적/정책성 문장(약관/고지)

### 1.3 ‘더보기/접기’ 트리거(수치)
- 본문: 3줄 초과 시 “더보기” 제공
- 리스트: 섹션 내 아이템이 8개 초과 시 “전체 보기” 제공

### 1.4 숫자/금액 표기의 일관 규칙(수치)
- 요약 화면에서 소수점:
    - 금액: 소수점 표시하지 않음
    - 퍼센트: 소수점 최대 1자리
    - 환율/단가: 소수점 최대 2자리
- 큰 수 축약:
    - 10,000 이상: 만 단위 축약(예: 12.3만)
    - 100,000,000 이상: 억 단위 축약(예: 1.2억)
    - 축약 표기 시 소수점 최대 1자리

### 1.5 데이터 없음/0/null 표준 문구(규칙)
- null(미입력): “아직 입력되지 않았어요”
- 0(의미 있는 값): “0”을 그대로 표시
- 비공개/권한 없음: “볼 수 없어요” + 사유 1줄

## 2) 검색/필터/정렬 — 탐색 UX 운영 규칙(정보 설계)
> SearchField 컴포넌트 자체의 Props가 아니라 “검색 기능 운영 기준”입니다.

### 2.1 검색 반응 시간(수치)
- 키 입력 → 결과 갱신:
    - 로컬 필터: 100ms 이내 목표
    - 서버 검색: 300ms 디바운스 + 응답 2초 이내 목표
- 2초 초과 시:
    - Skeleton/Loader + “검색 중” 라벨 표시

### 2.2 디바운스/요청 취소 규칙
- 디바운스 기본값: 300ms
- 연속 입력 중에는 이전 요청을 취소하고 마지막 요청만 반영
- 같은 쿼리 반복 요청은 10초 캐시

### 2.3 검색 결과 ‘없음’ 처리(규칙)
- 0건일 때는 반드시 2가지 중 1개 이상을 제공합니다.
    - 추천 쿼리(오타/띄어쓰기)
    - 필터 초기화 CTA
- “검색어 지우기” CTA는 0건 화면에서 항상 노출

### 2.4 필터 옵션 수 상한(수치)
- 1개 필터 그룹 내 옵션: 최대 7개
- 8개 이상이면:
    - “더보기” 또는 “검색 가능한 옵션 리스트”로 전환

### 2.5 필터 기본값 규칙
- 기본값은 “가장 많은 사용자가 찾는 상태” 1개만
- 다중 선택 필터는 기본값을 비우고, 대신 추천 필터를 상단 고정(최대 3개)

### 2.6 정렬 우선순위(규칙)
- 기본 정렬은 1개만(복합 정렬 금지)
- 추천 기본값:
    - 검색이 있는 경우: 관련도 → 최신순
    - 검색이 없는 목록: 최신순 또는 사용자에게 익숙한 규칙(가나다/최근 사용)

### 2.7 최근 검색/추천 키워드 운영(수치)
- 최근 검색 저장: 최대 10개
- 최근 검색 만료: 30일
- 추천 키워드: 최대 8개

## 3) 콘텐츠/미디어 UX — 업로드/미리보기/실패 재시도 규칙

### 3.1 파일 업로드 제한(권장 수치)
- 이미지:
    - 1장 최대 10MB
    - 긴 변(가로/세로) 최대 4096px(초과 시 리사이즈)
- 영상:
    - 1개 최대 200MB
    - 길이 최대 60초(초과 시 업로드 차단 또는 자르기 유도)
- 첨부 개수:
    - 한 게시물/요청당 최대 10개

### 3.2 업로드 상태 머신(최소)
- Queued → Uploading → Processing(필요 시) → Done / Failed

### 3.3 업로드 재시도(backoff) 기본값(수치)
- 자동 재시도: 최대 3회
- 간격: 1s → 2s → 4s
- 3회 실패 후:
    - 수동 재시도 버튼 노출
    - 실패 사유 1줄(네트워크/권한/용량)

### 3.4 미리보기/재생 규칙
- 자동 재생:
    - 기본 OFF
    - 단, “짧은 루프 미리보기”가 핵심이면 Wi‑Fi에서만 ON(모바일 데이터에서는 OFF)
- 썸네일 비율:
    - 기본 1:1
    - 문서/영수증 등 세로 콘텐츠는 3:4 허용

### 3.5 글(긴 문장/약관/공지) 접기 규칙(수치)
- 본문 6줄 초과 시 “접기/펼치기” 제공
- 약관/법적 고지 요약:
    - 상단 요약 문장 2줄 + “전문 보기” 링크

## 4) i18n 심화 — 문장 길이 팽창/레이아웃 붕괴 방지 규칙
> 토큰/타이포 자체가 아니라, 번역·표기·레이아웃 보호 규칙입니다.

### 4.1 텍스트 길이 가드레일(수치)
- 주요 버튼 라벨: 최대 8자(한글) 또는 최대 14자(영문)
- 보조 버튼/링크(TextButton): 최대 12자(한글) 또는 18자(영문)
- 탭 라벨: 최대 6자(한글) 또는 10자(영문)

### 4.2 줄바꿈/말줄임 규칙
- CTA 버튼:
    - 줄바꿈 금지(1줄 고정)
    - 초과 시:
        - 짧은 대체 문구로 교체(번역 키 분리)
- 리스트 제목:
    - 2줄까지 허용
    - 3줄 이상이면 말줄임 + 상세 화면으로 이동

### 4.3 숫자/통화/단위 포맷 규칙
- 통화:
    - 로컬 통화 기호 + 로컬 자릿수 구분을 사용
    - 요약 화면은 소수점 0자리(원화 기준)
- 퍼센트:
    - 0~1% 범위는 소수점 1자리 허용, 그 외는 정수 우선
- 날짜:
    - 요약: MM.DD 또는 YYYY.MM.DD
    - 상세: YYYY.MM.DD (요일)

### 4.4 번역 QA(수치)
- “긴 언어” 스트레스 테스트(독일어/러시아어/스페인어 가정):
    - 원문 대비 길이 1.3배를 견딜 것
- 폰트 확대(접근성) + 번역 길이 동시 조건에서:
    - 핵심 CTA가 화면 밖으로 밀려나지 않을 것

## 5) 기능 상태/데이터 모델 — CRUD/동기화/충돌을 제품 규칙으로 고정

### 5.1 기능별 최소 상태 세트(권장)
- Read(조회): Idle / Loading / Success / Empty / Error
- Write(저장): Editing / Submitting / Success / Failed
- Delete(삭제): Confirming / Deleting / Deleted / Failed

### 5.2 Optimistic update 허용/금지 규칙
- 허용(낮은 리스크):
    - 좋아요/북마크/라벨 변경
    - 개인 설정 토글
- 금지(높은 리스크):
    - 결제/송금/정산
    - 권한 변경/계정/보안 설정
    - 재고/좌석/예약 확정

### 5.3 충돌(Conflict) 처리 규칙
- 서버 버전과 로컬 편집이 충돌하면:
    - 자동 병합 금지(텍스트 문서가 아닌 이상)
    - 선택지 2개만 제공:
        1) 내 변경 적용(재시도)
        2) 서버 버전으로 되돌리기

### 5.4 저장/제출 버튼 활성화 기준(수치)
- 폼 유효성 검사:
    - 필수 필드 누락 시 CTA disabled
- 변경 감지:
    - 최초 값과 동일하면 CTA disabled
- 제출 중:
    - CTA는 loading + disabled 동시 적용

### 5.5 실패 분류(표준 코드 체계)
- 네트워크: OFFLINE / TIMEOUT
- 인증/권한: UNAUTHENTICATED / FORBIDDEN
- 검증: INVALID_INPUT
- 서버: SERVER_ERROR
- 각 실패는:
    - 사용자 문구 1줄
    - 다음 행동 1개(재시도/로그인/수정)

## 6) 최소 실행 경로(적용 순서)
1. 데이터 밀도 상한(카드 4개/텍스트 3줄/섹션 8개)을 UI 가드레일로 고정한다.
1. 검색 디바운스 300ms, 결과 2초 SLA, 0건 화면의 표준 CTA를 고정한다.
1. 업로드 제한(이미지 10MB/영상 200MB/60초)과 재시도(1→2→4, 3회)를 구현한다.
1. i18n 길이 가드레일(버튼 8자/14자)과 줄바꿈/말줄임 규칙을 적용한다.
1. CRUD 상태 세트와 optimistic update 금지 영역을 팀 규칙으로 고정한다.
