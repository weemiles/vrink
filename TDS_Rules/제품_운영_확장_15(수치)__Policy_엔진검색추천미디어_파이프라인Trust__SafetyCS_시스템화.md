## 1) 권한/정책(Policy) 엔진 설계 — 운영 규칙

### 1.1 정책 변경 안전장치
- [운영 가드레일] 정책 변경은 “즉시 전체 적용” 금지. 항상 2단계로 배포
    1) 내부(운영자/테스트 계정) 1일
    2) 실제 사용자 1% → 10% → 50% → 100% (각 단계 최소 2시간)
- [운영 가드레일] 정책 변경 1건당 롤백 버튼(이전 버전으로 스위치)은 필수

### 1.2 정책 버저닝/호환
- [운영 가드레일] 정책은 policy_version을 가진다(정수 증가)
- [운영 가드레일] 클라이언트는 최소 최근 2개 정책 버전을 이해해야 한다.
    - 모르는 rule은 “deny”가 아니라 “safe fallback(기능 축소)”로 처리

### 1.3 권한 모델 최소 단위
- [운영 가드레일] 권한 체크는 최소 3요소로 계산
    - actor(사용자/역할)
    - action(행동)
    - resource(대상)
- [운영 가드레일] 고위험 액션은 항상 2조건을 추가
    - risk_level >= high
    - step_up_auth = true (추가 인증)

### 1.4 캐시/일관성
- [운영 가드레일] 정책 평가 결과 캐시 TTL: 60초
- [운영 가드레일] 정책 업데이트 후 캐시 무효화 전파 목표: 5분 이내

## 2) 검색/추천 품질 운영(랭킹·튜닝·평가)

### 2.1 온라인 지표(최소 세트)
- [운영 가드레일] 검색/추천은 지표 4개를 고정
    - CTR(클릭률)
    - CVR(전환율, 정의 명시)
    - p95 응답 시간
    - 무결성/안전 가드레일(스팸/금칙 노출률)

### 2.2 성능 예산
- [운영 가드레일] 검색 API 응답 시간
    - p50 200ms 이하
    - p95 500ms 이하
- [운영 가드레일] 추천 피드 API 응답 시간
    - p50 300ms 이하
    - p95 800ms 이하

### 2.3 평가 샘플링/로그 보존
- [운영 가드레일] 검색 쿼리 로그 샘플링
    - 기본: 1%
    - 품질 이슈 기간: 5% (최대 7일)
- [운영 가드레일] 쿼리/클릭 로그 보존
    - 원문 쿼리: 30일
    - 집계(anon): 180일

### 2.4 랭킹 변경 롤아웃/롤백
- [운영 가드레일] 랭킹 변경은 A/B 또는 섀도우(비노출)로 24시간 검증 후 노출
- [운영 가드레일] 즉시 롤백 조건(하나라도)
    - CTR -5% 이상(상대)
    - CVR -3% 이상(상대)
    - p95 응답 +20% 이상
    - 금칙/스팸 노출률 +0.1%p 이상

### 2.5 스팸/어뷰징 방어(검색)
- [운영 가드레일] 동일 사용자의 검색 요청 레이트리밋
    - 10초: 최대 20회
    - 10분: 최대 300회
    - 초과 시 10분 쿨다운 + 캡차/추가 확인(가능하면)

## 3) 콘텐츠/미디어 파이프라인 운영

### 3.1 업로드 크기/형식 정책(기본)
- [운영 가드레일] 업로드 크기 상한
    - 이미지: 10MB
    - 동영상: 200MB
- [운영 가드레일] 업로드 실패 재시도
    - 자동 재시도: 최대 2회
    - 백오프: 2s → 10s

### 3.2 트랜스코딩/썸네일 SLA
- [운영 가드레일] 썸네일 생성 완료 목표: p95 60초 이내
- [운영 가드레일] 동영상 트랜스코딩 완료 목표: p95 10분 이내
- [운영 가드레일] SLA 위반 시 사용자 노출
    - 2분 초과: “처리 중” 배지
    - 10분 초과: “업로드는 완료, 처리 지연” 안내 + 재시도/문의 경로

### 3.3 검수/차단 운영
- [운영 가드레일] 자동 검수 결과가 high_risk면 즉시 비노출
- [운영 가드레일] 수동 검수 SLA
    - 1차 판정: 24시간 이내
    - 최종 판정: 72시간 이내

### 3.4 스토리지 비용/보관 정책
- [운영 가드레일] 미사용(0 view) 미디어 보관
    - 30일 후: 저해상도만 유지 또는 아카이브
- [운영 가드레일] 삭제 요청 반영
    - 사용자 삭제: 즉시 비노출 + 7일 내 물리 삭제

## 4) Trust & Safety (사기/어뷰즈) 운영

### 4.1 리스크 스코어 기반 정책(최소 3단계)
- [운영 가드레일] 리스크 레벨을 3단계로 고정
    - low: 정상
    - medium: 제한(일부 기능)
    - high: 차단 + 추가 인증

### 4.2 계정 탈취/봇 의심 시나리오 게이트
- [운영 가드레일] 아래 중 2개 이상이면 step-up 인증
    - 신규 기기 로그인
    - 10분 내 로그인 실패 5회 이상
    - 10분 내 민감 행동(비밀번호 변경/출금 등) 2회 이상
    - 비정상 지역/시간대 패턴

### 4.3 차단/해제 운영 SLA
- [운영 가드레일] 자동 차단은 최대 24시간까지만 유지(그 이후는 수동 검토)
- [운영 가드레일] 해제 요청 처리
    - 1차 응답 24시간 이내
    - 해제/유지 결정 72시간 이내

### 4.4 오탐(거짓 양성) 관리
- [운영 가드레일] 오탐률 목표: 0.5% 이하
- [운영 가드레일] 오탐률이 1%를 넘으면 룰/모델 업데이트를 중단하고 원인 분석 우선

## 5) 고객지원(CS) 도구·프로세스 시스템화

### 5.1 티켓 분류 체계(운영 최소 표준)
- [운영 가드레일] 모든 티켓은 최소 4속성을 가진다.
    - category(대분류)
    - severity(S1~S4)
    - status(open/in_progress/done)
    - owner(담당)

### 5.2 Severity 기준 + 응답 SLA
- [운영 가드레일] S1(결제/로그인 불가, 대규모 장애)
    - 1차 응답: 1시간 이내
    - 임시 완화: 4시간 이내
- [운영 가드레일] S2(핵심 기능 일부 장애)
    - 1차 응답: 4시간 이내
    - 해결/계획 공유: 24시간 이내
- [운영 가드레일] S3(불편/버그)
    - 1차 응답: 24시간 이내
- [운영 가드레일] S4(문의/요청)
    - 1차 응답: 48시간 이내

### 5.3 재발 방지 루프(운영 루틴)
- [운영 가드레일] 동일 카테고리 티켓이 7일 내 20건 이상이면
    - 제품/엔지니어링 이슈로 승격
    - “원인/재현/임시 완화/근본 해결/게이트 추가” 5항목을 문서화
- [운영 가드레일] 릴리즈 후 72시간은 VOC 모니터링 강화
    - CS 티켓/크래시/결제 실패를 4시간 간격으로 점검
