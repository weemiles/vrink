## 1) A11y(접근성) 검증/품질 게이트 — “릴리즈 차단” 규칙

### 1.1 릴리즈 차단 기준(Release blocking)
- [운영 가드레일] 아래 항목 중 1개라도 실패하면 릴리즈 차단
    - 터치 타겟 최소 44×44px 위반[1]
    - 주요 인터랙션 요소에 접근성 라벨 누락(아이콘만 있는 버튼 포함)[2]
    - 텍스트 대비(일반 텍스트) 4.5:1 미만
    - “더 큰 텍스트” 설정에서 핵심 CTA가 화면 밖으로 밀려 접근 불가

### 1.2 스크린리더(VoiceOver/TalkBack) 최소 체크리스트
- [운영 가드레일] 핵심 플로우(온보딩/결제/로그인/설정)는 최소 1회/릴리즈 스크린리더 점검
- [운영 가드레일] 포커스 순서 규칙
    - 상단 → 하단, 좌 → 우, 논리 흐름 우선
    - 모달/바텀시트 오픈 시 포커스는 200ms 이내 내부 첫 요소로 이동
    - 닫힘 시 포커스는 “열었던 트리거”로 복귀

### 1.3 동작 줄이기(Reduce motion)
- [운영 가드레일] Reduce motion ON이면
    - 자동 재생 애니메이션/루프는 기본 OFF
    - 화면 전환 모션 duration을 최대 150ms로 제한

## 2) Theming(다크모드/고대비) 운영 규칙

### 2.1 토큰 사용 강제
- [운영 가드레일] 색상/배경/보더는 토큰만 허용
    - 하드코딩 HEX/RGB 금지
    - 예외 허용 시: 코드 리뷰에서 사유/대체 토큰을 명시

### 2.2 다크모드 회귀(Regression) 게이트
- [운영 가드레일] 릴리즈 전 “라이트/다크” 2모드 스냅샷 비교를 최소 10개 핵심 화면에서 수행
- [운영 가드레일] 다크모드에서 아래 현상 발생 시 차단
    - 텍스트 대비 4.5:1 미만
    - divider/border가 배경과 구분 불가(시각적 경계 소실)
    - disabled와 enabled의 구분이 불가(상태 인지 실패)

### 2.3 고대비(High contrast) 정책
- [운영 가드레일] 고대비 모드(지원 가능 환경)에서
    - 정보 전달이 색상 하나에만 의존하면 실패
    - 색상 + 아이콘/텍스트/패턴 중 최소 1개를 추가

---

## 3) Motion(애니메이션) 시스템 운영 — 시간/빈도/중단 규칙

### 3.1 Duration 표준(권장 범위)
- [운영 가드레일] 마이크로 인터랙션(pressed, toggle)
    - 80~120ms
- [운영 가드레일] 화면 전환(페이지 push/popup)
    - 180~250ms
- [운영 가드레일] 로딩/진행(루프)
    - 1사이클 800~1200ms

### 3.2 반복/주기 제한
- [운영 가드레일] 동일 요소의 “자동 강조(펄스/바운스)”는
    - 최초 진입 후 최대 3회까지만
    - 그 이후 자동 반복 금지

### 3.3 중단(Interrupt) 규칙
- [운영 가드레일] 사용자가 스크롤/탭/입력을 시작하면
    - 진행 중인 장식 애니메이션은 즉시 중단 또는 opacity 0으로 페이드

---

## 4) i18n(번역/문장 길이) + 레이아웃 붕괴 방지 규칙

### 4.1 길이 증가율 가정(기본)
- [운영 가드레일] 번역 길이 증가율을 UI 설계 기본값으로 고정
    - 영어 → 독일어/러시아어: 최대 1.3×
    - 영어 → 프랑스어/스페인어: 최대 1.2×
    - 한국어 → 영어: 최대 1.5×

### 4.2 한 줄 컴포넌트(버튼/탭/배지) 정책
- [운영 가드레일] 한 줄만 허용되는 컴포넌트는 아래 중 1개를 반드시 선택
    1) 말줄임(ellipsis)
    2) 짧은 별칭 키(예: cta.save.short)
    3) 아이콘 + 짧은 텍스트
- [운영 가드레일] 탭/세그먼트 라벨은 목표 6~8자, 최대 12자
    - 초과하면 약어 키로 분기 또는 UI 구조 변경

### 4.3 날짜/숫자 포맷의 지역화
- [운영 가드레일] 표시 포맷은 locale 기반으로만 출력
    - 하드코딩 YYYY.MM.DD 고정 금지
    - 통화 기호 위치(앞/뒤)는 로케일을 따른다

## 5) QA 매트릭스 — 상태×환경 조합 테스트(최소 세트)

### 5.1 테스트해야 하는 “상태” 최소 6종
- [운영 가드레일] 데이터 화면 기준
    - initial loading
    - loaded
    - empty
    - error retryable
    - error blocking
    - offline/stale

### 5.2 테스트해야 하는 “환경” 최소 6종
- [운영 가드레일] 환경 축
    - 라이트/다크(2)
    - 텍스트 스케일 100% / 200%(2)
    - 네트워크 정상 / 느림 / 오프라인(3)
    - Reduce motion ON/OFF(2)
    - 권한 허용/거부(예: 알림/위치/카메라)(2)
    - 언어: KO + EN(최소 2개)

### 5.3 최소 조합 규칙(폭발 방지)
- [운영 가드레일] 모든 조합을 곱으로 돌리지 말고 “대표 조합 12개”로 고정
    - 핵심 플로우 3개 × 환경 4개 = 12개
    - 환경 4개 기본 세트
        1) Light + 100% + 정상 네트워크
        2) Dark + 100% + 정상 네트워크
        3) Light + 200% + 정상 네트워크
        4) Light + 100% + 오프라인(stale)

---

## 6) 최소 적용 순서(실무 도입)
1) A11y 릴리즈 차단 기준(1.1)부터 “체크 실패 시 배포 불가”로 고정
2) 다크모드 회귀 게이트(2.2) 10개 핵심 화면 스냅샷 비교
3) Motion duration 표준(3.1) 적용 + 반복 3회 제한(3.2)
4) i18n 길이 증가율(4.1)과 탭/라벨 최대 12자(4.2)로 레이아웃 규칙 고정
5) QA 대표 조합 12개(5.3)로 테스트 루틴 고정
